RL Kill Switch — 자율운항 안전 차단 노드

강화학습 정책으로 제어되는 자율 예인선(tug1)의 ROS2 안전 레이어. 추론 노드와 Arduino 사이에 위치하며, 6가지 트리거 조건을 실시간 감시하여 이상 감지 시 즉시 PWM 출력을 [0.0, 0.0] 으로 차단하고 Arduino 시리얼 통신을 단독으로 관리한다.

01 개발 배경 및 문제 정의
배경

충남대 IMRL 연구실은 33 × 33 m 실내 수조에서 강화학습 정책 네트워크(TorchScript JIT, 44-dim 관측 벡터)로 제어되는 자율 예인선 실험을 수행한다. 기존 시스템은 추론 노드가 PWM 명령을 Arduino에 직접 전송하는 단일 파이프라인 구조였다.

문제: RL 정책이 비정상 출력을 생성하거나 GPS/IMU 센서가 열화될 경우, RC 조종기 연결이 끊어지거나 선박이 수조 경계를 이탈해도 이를 차단할 독립적인 안전 레이어가 없었다.
추가 제약: 기존 추론 노드가 Arduino 시리얼 포트를 직접 점유하고 있어, Kill Switch가 중간에 개입해도 Arduino는 정책 출력을 그대로 수신할 수 있었다.
목표 및 설계 방향
1
토픽 분리: 추론 노드 출력을 /tug1/PWM_raw로 변경하고 Kill Switch가 이를 수신·검사 후 /tug1/PWM으로 relay — Arduino 직접 접근 차단.
2
시리얼 독점: Arduino 시리얼 통신을 Kill Switch로 이전. 추론 노드의 send_to_arduino() 및 serial.Serial 초기화 코드 제거.
3
6가지 트리거: 경계이탈(T1), RC 두절(T2), 원격통신 두절(T3), GPS 불량(T4), PWM 범위(T5), IMU 불량(T6) 독립 감시.
4
복구 분류: 수동 트리거(T1·T2·T5)는 ROS2 서비스 콜로만 복구, 자동복구 트리거(T3·T4·T6)는 조건 해소 시 자동 정상화.

03 핵심 구현 상세
FrequencyMonitor — 슬라이딩 윈도우 주파수 감시

ROS2 센서 콜백 주파수를 실시간 측정하는 독립 유틸리티 클래스. GPS(3 Hz)와 IMU(30 Hz) 각각에 인스턴스를 생성하여 독립적으로 감시한다.

class FrequencyMonitor:
    def __init__(self, window_sec=1.0, min_hz=3.0, consecutive_fail=3):
        self._ts = deque()        # timestamp 슬라이딩 윈도우
        self.fail_count = 0

    def update(self) -> bool:
        now = time.time()
        self._ts.append(now)
        while self._ts and (now - self._ts[0]) > self.window_sec:
            self._ts.popleft()        # 윈도우 밖 타임스탬프 제거
        hz = len(self._ts) / self.window_sec
        if hz < self.min_hz:
            self.fail_count += 1
        else:
            self.fail_count = 0    # 회복 시 카운터 리셋
        return self.fail_count >= self.consecutive_fail
T1 — Ray-Casting 경계이탈 감지

수조 경계 GPS 좌표를 노드 시작 시 한 번만 로컬 탱크 좌표로 변환하여 저장. GPS 콜백마다 Ray-casting으로 다각형 내부 여부 판별.

설계 결정: ROS2 파라미터는 중첩 리스트를 지원하지 않으므로 boundary_gps_points를 평탄화된 1-D float 배열 [lat1, lon1, lat2, lon2, ...]로 선언하고 노드 내부에서 쌍으로 파싱. 
def _point_in_polygon(self, px, pz, polygon) -> bool:
    inside = False
    n, j = len(polygon), len(polygon) - 1
    for i in range(n):
        xi, zi = polygon[i]
        xj, zj = polygon[j]
        if ((zi > pz) != (zj > pz)) and \
           (px < (xj - xi) * (pz - zi) / (zj - zi) + xi):
            inside = not inside
        j = i
    return inside
T3 — ICMP Ping 원격통신 감시

ROS 토픽과 무관하게 USV가 직접 Main PC에 1 Hz ICMP ping을 보내 통신 상태를 독립적으로 확인. 백그라운드 daemon 스레드로 실행.

def _ping_once(self, host: str) -> float | None:
    result = subprocess.run(
        ['ping', '-c', '1', '-W', '1', host],
        capture_output=True, text=True, timeout=2.0
    )
    if result.returncode == 0:
        for token in result.stdout.split():
            if token.startswith('time='):
                return float(token.split('=')[1])  # RTT (ms)
    return None  # 응답 없음

좌표계 변환 — WGS84 → UTM → Tank-LHS
# WGS84 → UTM (Python utm 라이브러리)
E0, N0, *_ = utm.from_latlon(ORIGIN_LAT, ORIGIN_LON)

def enu_to_tank(e, n, u):
    # Tank-LHS: +X=East, +Y=Down, +Z=North
    return (e - E0, -u, n - N0)

# GPS 콜백에서 사용
e, n, *_ = utm.from_latlon(msg.latitude, msg.longitude)
x, _, z = enu_to_tank(e, n, 0.0)  # Kill Switch와 Inference Node 동일 변환
일관성 보장: Kill Switch와 Inference Node가 동일한 ORIGIN_LAT/LON과 enu_to_tank() 함수를 사용하여 경계 좌표 변환 오차 없음.

알려진 한계 및 향후 과제
하드코딩된 좌표: boundary_gps_points와 main_pc_ip는 배포 전 kill_switch_params.yaml에서 실측값으로 교체 필요.
센서 손실 처리 없음: GPS/IMU 토픽이 완전히 중단되면 FrequencyMonitor가 정확한 Hz를 계산하지 못하고 즉시 트리거가 발동되지 않을 수 있음 (콜백 미호출 시 update() 미실행).
향후 개선: 주기적인 Watchdog 타이머를 추가하여 콜백이 전혀 없을 경우에도 T4/T6 트리거를 강제 발동하도록 개선 예정.
